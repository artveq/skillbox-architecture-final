# Анализ стейкхолдеров и их интересов

| Стейкхолдер | Роль в проекте | Интересы | Потенциальные риски / опасения | Влияние на архитектуру |
|-------------|----------------|----------|--------------------------------|--------------------------|
| **Менеджмент компании** | Стратегические решения, бюджет | Рост продаж; укрепление бренда; глобальная экспансия | Высокая стоимость; долгий time-to-market | Требование масштабируемости; интеграция с e-commerce; поддержка промо |
| **Маркетинг** | Контент, промо, коммуникации | Персональные промо; аналитика поведения; рост вовлечённости | Недостаточная персонализация; низкий CTR | Promo Service; Recommendation Service; аналитическая платформа |
| **Пользователи (спортсмены, любители, дети)** | Основные потребители продукта | Удобство; мотивация; социальные функции; точность данных | Сложность интерфейса; низкая точность датчиков; задержки | UX-требования; низкая задержка API; интеграции с устройствами |
| **Профессиональные спортсмены** | Ролевые модели, источник данных | Корректное отображение результатов; сравнение; безопасность тренировок | Некорректные данные; репутационные риски | Достоверность аналитики; валидация данных |
| **Разработчики** | Создание и поддержка системы | Чёткая архитектура; стандарты; независимость сервисов | Сложность мультиоблака; разрозненные команды | Микросервисы; CI/CD; стандартизация API |
| **DevOps / SRE** | Надёжность, инфраструктура | Предсказуемая нагрузка; автоматизация; наблюдаемость | Пиковые нагрузки; сложность мониторинга | Event-driven; autoscaling; логирование и трассировка |
| **Партнёры (производители устройств)** | Интеграции с датчиками | Совместимость; стабильная передача данных | Разные протоколы; нестабильные каналы | IoT Gateway; стандартизированные SDK |
| **Региональные представители** | Локальный маркетинг | Региональные акции; локальный контент | Несоответствие локальным законам; культурные различия | Локализация; геораспределённые данные |
| **Юридический отдел** | Соответствие законам | GDPR; защита данных; хранение по регионам | Нарушение законодательства; штрафы | Security Zone; региональные БД; шифрование |
| **Команда аналитики / Data Science** | ML-модели, рекомендации | Доступ к данным; качество данных; масштабируемость | Недостаток данных; задержки; несогласованность | Analytics Zone; ETL; event sourcing |
| **Команда e-commerce** | Продажи, каталоги | Интеграция каталога; персональные рекомендации | Несовместимость API; задержки | Internal API Gateway; Inventory Service |
| **Команда поддержки пользователей** | Работа с жалобами | Простые инструменты модерации; прозрачность данных | Высокий объём жалоб; токсичность в соц. группах | Модерация контента; логирование; инструменты администрирования |
