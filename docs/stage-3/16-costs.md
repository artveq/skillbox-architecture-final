# 16. Стоимость владения системой в первый, второй и пятый годы

## 1. Подход к оценке стоимости

Оценка стоимости владения (TCO) выполнена на основе:

- разделения затрат на **CAPEX** (разовые вложения) и **OPEX** (ежегодные расходы);
- роста пользовательской базы (MAU/DAU);
- роста объёма данных (телеметрия, социальные события, аналитика);
- увеличения количества регионов;
- нагрузки на доменные сервисы и инфраструктуру.

Все значения приведены в условных единицах (**у.е.**) для удобства сравнения.


## 2. CAPEX (разовые вложения в 1-й год, MVP)

CAPEX включает создание платформы, доменных сервисов, инфраструктурного шаблона региона и запуск первых регионов.

| Статья | Стоимость (у.е.) | Комментарий |
|--------|------------------|-------------|
| Разработка платформы | **1 200** | Базовая архитектура, API, ядро |
| Доменные сервисы | **800** | User, Training, Social, Gamification, Inventory |
| Инфраструктурный шаблон региона | **400** | Edge, сервисный слой, данные, безопасность |
| Запуск первых 2 регионов | **600** | Развёртывание, настройка, тестирование |
| **Итого CAPEX** | **3 000 у.е.** | Разово в первый год |

## 4. Рост объёма данных

### Источники данных:
- телеметрия тренировок (основной объём);
- социальные события (фид, лайки, связи);
- аналитические агрегаты и ML-фичи;
- исторические данные.

| Год | Объём данных в регионе | Всего по системе | Комментарий |
|-----|------------------------|------------------|-------------|
| **1** | 5–10 ТБ | 10–20 ТБ | Начальная история |
| **2** | 20–40 ТБ | 60–120 ТБ | Рост активности |
| **5** | 150–300 ТБ | 750–1500 ТБ | Большие исторические данные |

## 5. OPEX (ежегодные расходы)

### Структура OPEX:
1. Инфраструктура регионов  
2. Хранение данных  
3. Сетевой трафик  
4. Наблюдаемость и эксплуатация  
5. Развитие платформы
