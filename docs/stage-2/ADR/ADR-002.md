# ADR-002: Стиль взаимодействия сервисов — REST + event-driven

## Контекст
Сервисы должны поддерживать как синхронные запросы (API), так и асинхронные реакции (лента, аналитика, челленджи).

## Решение
Использовать гибридный подход:
- REST для команд и запросов,
- Event Bus для событий и асинхронных процессов.

## Альтернативы
- Только REST — жёсткие зависимости, каскадные сбои.
- Только события — сложно реализовать запросы/команды.

## Обоснование
- Соответствует критическим сценариям (лента, челленджи, аналитика).  
- Уменьшает связность сервисов.  
- Поддерживает масштабируемость и надёжность.

## Последствия
- Требуется единый формат событий.  
- Необходима система мониторинга очередей и шины.
