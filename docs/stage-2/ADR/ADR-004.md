# ADR-004: Стратегия хранения данных — региональные сторы + глобальный Data Lake

## Контекст
Платформа работает в разных странах с разными законами о данных.  
Нужны низкие задержки и глобальная аналитика.

## Решение
Использовать:
- региональные хранилища для оперативных данных,
- глобальный Data Lake / DWH для аналитики.

## Альтернативы
- Единое глобальное хранилище — проблемы с задержками и compliance.
- Полностью региональная архитектура — сложно строить глобальную аналитику.

## Обоснование
- Соответствие GDPR и локальным законам.  
- Низкая задержка для пользователей.  
- Возможность глобальной аналитики и ML.

## Последствия
- Необходима система репликации и ETL.  
- Усложнение управления схемами данных.
