# ADR-005: Интеграция устройств — использование IoT Gateway

## Контекст
Устройства используют разные протоколы (BLE, MQTT, HTTPS), имеют нестабильные каналы связи и требуют буферизации телеметрии.

## Решение
Использовать выделенный IoT Gateway как единую точку входа для устройств.

## Альтернативы
- Прямое подключение к сервисам — дублирование логики, проблемы безопасности.
- Прокси-слой без IoT‑специализации — не покрывает особенности устройств.

## Обоснование
- Централизация протоколов и аутентификации.  
- Буферизация и QoS.  
- Соответствие критическому сценарию “запись тренировки”.

## Последствия
- Требуется отдельная команда/подсистема для IoT.  
- Усложнение инфраструктуры.
