# ADR-001: Выбор архитектурного стиля — микросервисы

## Контекст
Платформа должна поддерживать глобальный рост, высокую нагрузку, независимые домены (Training, Social, Gamification, Analytics) и параллельную работу нескольких команд.  
Критические сценарии (телеметрия, лента, челленджи) имеют разные профили нагрузки.

## Решение
Использовать доменно-ориентированную микросервисную архитектуру.

## Альтернативы
- Модульный монолит — проще, но плохо масштабируется и усложняет независимую разработку.
- SOA — слишком тяжёлый и не даёт нужной гибкости.

## Обоснование
- Независимое масштабирование доменов.  
- Изоляция отказов.  
- Поддержка event-driven взаимодействия.  
- Соответствие NFR по масштабируемости и надёжности.

## Последствия
- Усложнение DevOps и observability.  
- Необходимость стандартизации API и событий.
