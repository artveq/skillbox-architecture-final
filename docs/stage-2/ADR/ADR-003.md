# ADR-003: Сегментация API — External, Internal и IoT Gateway

## Контекст
Разные типы клиентов (мобильные приложения, внутренние сервисы, устройства) имеют разные требования по безопасности, нагрузке и протоколам.

## Решение
Использовать три независимых API-шлюза:
- External API Gateway,
- Internal API Gateway,
- IoT Gateway.

## Альтернативы
- Один общий шлюз — смешение контуров, сложность безопасности.
- Два шлюза (external/internal) — не покрывает IoT-нагрузку.

## Обоснование
- Соответствует концептуальной архитектуре.  
- Упрощает безопасность и rate limiting.  
- Позволяет эволюционировать API независимо.

## Последствия
- Увеличение числа точек управления.  
- Требуется единая политика аутентификации.
